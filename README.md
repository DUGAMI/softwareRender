# SoftwareRenderer

2025년 1월부터 컴퓨터 그래픽스에 대한 이해를 깊이기 위해 소프트웨어 렌더러를 직접 만들기로 했습니다.
코드를 작성한 곳은 군대입니다... 맞습니다, 저는 이미 11개월째 복무 중이며, 현재 복무 진행도는 65%입니다. 전역할 때까지 간단한 게임 개발 엔진을 구현하고 싶습니다. 이를 통해 간단한 게임을 만들 수 있거나, 한 발 물러서서 간단한 3D 씬 에디터라도 만들어보고자 합니다.
개발 언어로 JavaScript를 선택한 이유는, 군대 컴퓨터에 프로그래밍 소프트웨어를 설치할 수 없어 온라인 개발 환경을 사용할 수밖에 없었기 때문입니다. 그런데 온라인 환경에서는 별도의 창을 열어 무언가를 렌더링하기가 어려워, 고민 끝에 JavaScript를 선택했습니다. 참고 자료로는 GAMES101을 사용했으며, 이해가 안 되는 부분은 ChatGPT에게 물어보거나 블로그를 검색했습니다.

### 2025.9.16

현재 복무 진행도는 90%입니다. 소프트웨어 렌더러에 필요한 기본 기능들은 어느 정도 구현했다고 할 수 있고, 이제 남은 작업은 계속 보완하면서 PBR, 레이 트레이싱 같은 고급 조명 모델을 공부하는 것입니다. 솔직히 말하면 효율이 꽤 낮았습니다. 4월부터 지금까지 기본 기능 구현 외에는 대부분 리팩토링에 시간을 쏟았습니다. 처음에는 코드가 꽤 엉망이었는데, 지금은 그나마 좀 봐줄 만해졌습니다.
시간이 남는다면 WebGL을 GPU 렌더링 백엔드로 도입해 프로그램의 실용성을 높이는 것도 고려해볼 생각입니다.

### 실행 방법

파일 업로드 항목이 두 개 있습니다. 왼쪽에는 렌더링할 OBJ 파일을, 오른쪽에는 텍스처 파일을 업로드하면 됩니다.
写代码的场所是军队... 没错，我已经服役11个月了，现在服役进度是65%，我希望我能在服役结束时把一个简单的游戏开发引擎实现出来，你可以用它来开发简单的游戏，或者退一步，只是开发一个简单的3d场景编辑器。

我选择使用javascript来开发这个项目，原因是军队的电脑没法安装编程软件，我不得不使用线上编程环境，但是线上编程环境无法打开一个窗口，来渲染一些东西，思来想去，还是用javascript吧。项目的参考资料使用使用games101，有不明白的地方就去问chatgpt或者去搜索一些博客。
